# 🗄️ NAS 共享目錄規範（科定AI）

## 掛載路徑對照

| 容器內路徑 | NAS 來源 | 權限 | 用途 |
|-----------|----------|------|------|
| `/mnt/kd-share` | `\\192.168.1.218\KD共用` (頂層) | **唯讀** | 讀取公司共用檔案 |
| `/mnt/kd-dc` | `\\192.168.1.218\KD共用\18_各部門共享區\21_系統開發課\89.KedingDC` | **可讀寫** | 科定AI 專屬工作區 |

## Windows 路徑轉換

使用者貼 Windows 路徑時，自動轉換：
- `\\192.168.1.218\KD共用\...` → `/mnt/kd-share/...`
- `Z:\...` → `/mnt/kd-share/...`

轉換時將 `\` 替換為 `/`，去掉前綴 `\\192.168.1.218\KD共用` 或 `Z:`。
**不需要問使用者確認，直接轉換使用。**

## ⚠️ 絕對規則

### `/mnt/kd-share`（KD共用全區）

| 操作 | 允許？ | 說明 |
|------|--------|------|
| 讀取使用者指定的完整路徑檔案 | ✅ | 使用者貼路徑，你去讀 |
| `find`、`ls -R`、遞迴搜尋 | ⛔ **禁止** | 檔案數量極龐大，會造成系統負載 |
| `ls` 列出某層目錄 | ⚠️ 僅限使用者明確要求時 | 最多列一層，禁止遞迴 |
| 寫入任何檔案 | ⛔ **禁止** | 唯讀掛載，即使嘗試也會失敗 |
| `grep -r` 搜尋檔案內容 | ⛔ **禁止** | 同上，龐大目錄 |

### `/mnt/kd-dc`（89.KedingDC）

| 操作 | 允許？ | 說明 |
|------|--------|------|
| 讀取、搜尋 | ✅ | 自由使用 |
| 寫入、建立檔案 | ✅ | 產出物、報告、暫存都放這裡 |
| `find`、`ls -R` | ✅ | 此目錄量可控 |

## 使用流程

```
使用者說：「幫我看 Z:\18_各部門共享區\05_業務部\報價單.xlsx」

→ 轉換路徑：/mnt/kd-share/18_各部門共享區/05_業務部/報價單.xlsx
→ 直接讀取該檔案
→ 如需產出分析結果，寫到 /mnt/kd-dc/ 下
```

## 錯誤處理

- 如果路徑不存在：告知使用者「找不到該檔案，請確認路徑是否正確」
- **永遠不要**嘗試用搜尋來「幫忙找」使用者可能放錯的檔案位置
- **永遠不要**嘗試對 `/mnt/kd-share` 寫入後回報「權限不足」——你本來就不該寫
