## 一、專案目標

將科定內部產品知識 Q&A，透過 Agent 平台 Workflow 自動生成六格彩色漫畫，加上品牌框後推播至企業微信，作為員工教育訓練素材。

### 核心價值（vs 直接用 ChatGPT）

| 痛點 | 我們的優勢 |
|------|-----------|
| 每次都要手動寫很長的提示詞 | 主管只需貼 Q&A 原文，系統自動分鏡 |
| 風格不一致 | 預設固定品牌風格 prompt，每次輸出統一 |
| 生成後要自己加 logo/品牌框 | 自動加品牌框 |
| 要自己想怎麼拆格分鏡 | AI 依固定結構自動拆解 |
| 沒有審核流程 | 主管確認才推播 |
| 推播要手動操作 | 確認後一鍵推播 |
| 無法累積優化 | 企業形象元素逐步補充到 prompt |

---

## 二、流程

```
主管輸入 Q&A 文字
    ↓
AI 根據固定分鏡結構自動產生六格分鏡腳本
    ↓
AI 根據分鏡生成六格彩色漫畫
    ↓
自動加上科定品牌框
    ↓
主管確認（不滿意可重新生成）
    ↓
Agent 調用推播到企微
```

---

## 三、輸入格式

主管手動在 Agent 平台 Workflow 輸入，結構如下：

```
產品線: [產品線名稱]
問題分類: [分類]
問題: [問題標題]
回覆: [完整回覆內容]
```

### 真實範例

```
產品線: 房間門
問題分類: 產品細節
問題: 房間門丈量遇牆面破口/斷差處理原則
回覆: 房間門丈量如遇牆面破口、斷差等問題，請依現場狀況分別處理。
涉及額外工序者需加收費用，並於丈量時先行向客戶說明與留存紀錄。

1、特殊牆面：請客戶自行處理。例如藝術漆、貼膜/壁紙、特殊漆色等，須由原牆面工班修補。
2、破口8mm以下：原則由客戶自行填縫收尾。如客戶堅持我司處理，加收2,000元/樘。
3、破口大於8mm：由客戶先行整平，後續安排復丈（復丈加收3,000元/趟）。如還有牆面歪斜，由我司工班補板填縫，加收1,000元/樘。
4、浴室磁磚斷差：門框耳朵部位裁切加工加收700元/樘。
```

## 五、六格分鏡框架

### 共通規則（所有框架適用）

1. **Step 0**：分鏡前先提取本題「不可遺漏的 3-5 個關鍵訊息」
2. 分鏡完成後**自我檢查**：每個關鍵訊息是否都有對應到至少一格，遺漏就調整
3. 文字一律繁體中文
4. 場景根據題目內容自動判斷

### 框架 A：產品細節 SOP 類

**適用**：有明確操作流程、收費標準的題目

| 格 | 結構 | 目的 |
|----|------|------|
| 1 | 了解背景 | 這個知識的由來/為什麼重要 |
| 2 | 現場痛點 | 客戶/師傅實際遇到的問題場景 |
| 3 | 應對話術一 | 面對客戶時的專業說明 |
| 4 | 應對話術二 | 坦承限制 + 公司處理方式 |
| 5 | 具體 SOP / 收費標準 | 操作細節 |
| 6 | 專業總結 | 核心原則 + 建立信任收尾 |

**比稿範例**：房間門丈量遇牆面破口/斷差處理原則

---

### 框架 B：客訴應對類（產品改善型）

**適用**：產品有已知問題、公司已改善，需要員工知道怎麼解釋給客戶

| 格 | 結構 | 目的 |
|----|------|------|
| 1 | 了解產品背景 | 建立員工信心 |
| 2 | 客訴痛點呈現 | 客戶在抱怨什麼 |
| 3 | 話術示範 — 解釋設計初衷 | 說明產品為什麼這樣設計 |
| 4 | 話術示範 — 坦承盲點 + 已優化 | 公司已改善的事實 |
| 5 | 改善成果展示 | 具體做了什麼改善 |
| 6 | 總結 | 雙贏收尾 + 專業形象 |

**比稿範例**：強力膠過稀

---

### 框架 C：對話示範類（情境應變型）

**適用**：重點在「一來一回怎麼接話」，強調情緒節奏和具體步驟

| 格 | 結構 | 目的 |
|----|------|------|
| 1 | 客訴爆發 | 客戶生氣的瞬間 |
| 2 | 同理接住 | 業務第一反應：穩住情緒 |
| 3 | 話術推進 — 蒐集/釐清 | 引導客戶提供資訊或釐清原因 |
| 4 | 話術推進 — 提出解方 | 主動解決客戶現場問題 |
| 5 | 後續跟進 / 施工教學 | 收尾動作或正確做法教學 |
| 6 | 專業總結 | 處理原則歸納 + 建立信任 |

**比稿範例**：底材脫離、美耐板黏貼不佳

---

### 框架選用判斷

| 題目特徵 | 選用框架 |
|----------|----------|
| 有明確操作流程、分級處理、收費標準 | A |
| 產品有已知缺點、公司已改善、需解釋設計初衷 | B |
| 重點在對話節奏、情緒應對、一來一回示範 | C |
| 簡單 Q&A，一個問題 + 說明，無對話情境 | D |

---

### 框架 D：產品知識圖解（單圖）

**適用**：單純的 Q&A（一個問題 + 說明），沒有對話情境，重點是用圖像輔助記憶

| 項目 | 規格 |
|------|------|
| 格式 | 單張圖（非分格） |
| 尺寸 | 1024x1024 |
| 風格 | 資訊圖表風格，乾淨清楚，配色柔和專業 |
| 文字 | 繁體中文，只放關鍵字眼，字數少 |
| 品牌框 | 加（AddBrandFrame，**header = dark**） |

**比稿範例**：批批板貼合膨拱原因與注意事項、地板施工配合方式、房間門細部規格

---

### 品質原則：圖片真實感

圖片中出現的產品/工具，外觀必須貼近現實（形狀、配色、比例），讓行業內的人一眼認得出來。

- ❌ 用 AI 預設的泛用圖像（如亂畫的膠水瓶）
- ✅ 根據實際產品特徵描述（形狀、顏色、材質）
- 品牌名稱不直接露出，但外型意象要像
- 產品狀態描述要精確（如膨拱 = 表面水泡狀隆起，不是板材彎曲）

### 品質原則：畫面簡潔

- 不要一張圖塞太多資訊，寧可留白
- 關鍵字精簡，不堆文字
- 需要區分層次時（如標配 vs 選配），用分區塊呈現

### 品質原則：品牌露出

- 制服上寫「KD」字母（白色），不畫 logo 圖形
- 產品桶罐等只用外型意象（形狀+配色），不放任何品牌文字
- 科定品牌由品牌框（AddBrandFrame）統一呈現

### 品牌框設定

- **預設使用 `header=dark`（黑框）**，不需要每次判斷
- dark = 深色底的 header（目前插畫都搭配黑框）
- light = 淺色底的 header（暫不使用）

### 交辦流程

1. 章魚哥負責交辦和品質把關，**不自己生圖**
2. 海綿寶寶負責生圖（GenerateImage）+ 加框（AddBrandFrame）+ 放到 drop
3. 生圖前執行 Step 0：提取關鍵訊息，確保不遺漏重點
4. 啟銘（UID: 1511580116352897094）負責確認插畫品質，提供調整意見
