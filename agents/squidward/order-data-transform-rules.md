# 訂購資料轉換規則

## 品號組成規則

### P、C、N 系列

**品號結構：`系列` + `規格1` + `色號` + `規格2`（規格2 視規格1 而定）**

- 規格1 為 A、B 時：`系列` + `規格1` + `色號`（無規格2）
- 規格1 為 M、G、V 時：`系列` + `規格1` + `色號` + `規格2`

### K 系列（塗裝/天然木皮板）

**品號結構：`K` + `產品分類(1碼)` + `木種(3碼)` + `色碼/表面處理` + `拼法(天然才有)`**

**第2碼 — 產品分類：**

| 第2碼 | 產品分類 |
|-------|---------|
| 4 | 未漆木皮板 |
| 6 | 塗裝木皮板 |
| 7 | 未漆不織布 |
| 9 | 塗裝不織布 |

**木皮板 → 不織布轉換：** 客人說「布」「不織布」→ 第2碼改為不織布代碼
- 塗裝：6 → 9（如 K6176AS → K9176AS）
- 未漆：4 → 7（如 K4176AS → K7176AS）

直接原樣輸出品號，僅在客人要「布/不織布」時轉換第2碼。

### L 系列

**品號結構：`L` + `色號`（無規格1、規格2）**

直接原樣輸出，不需轉換。

## 系列轉換規則

- **N 系列沒有 G（自黏）和 V（ABS）**：N 系列遇到自黏或 ABS 需求 → 品號系列改用 C
  - 例：N8102 的自黏 → CG810219、CG8102A、CG8102X48
  - 例：N8102 的 ABS → CV810222

- **N 系列沒有 NM+1（單面木芯板）**：N 系列目前沒有單面木芯板，遇到時提醒使用者確認

- **P → L 轉換**：客人給 P 系列色號，但明確說「美耐板」→ 品號改為 L+色號
  - 例：P160F美耐板 → L160F
  - ⚠️ **前提：L 系列必須有該色號才能轉換**，P/L/C/N 色號不互通，不能硬轉

- **C → L 轉換**：同上邏輯，C 系列色號+美耐板 → L+色號，但要確認 L 有該色號

- **L + 規格 = 不確定**：客人給 L 系列色號，但有規格1（1分、2分、麻等）→ 資訊不確定，需請業務跟客人確認是要 P 還是 L，不能硬轉換

- **K 系列默認1分板**：K 系列沒有分規格，品號直接原樣輸出。如客人有提2分、6分等字眼為訂製品。

## 「28」判斷邏輯

- 有「厚板、6分、麻、板、木芯、木心」等關鍵字 + 28/2*8 → M 的 128/228
- 無上述關鍵字，單獨出現「28」「2*8」→ G+A（自黏2尺2.44米）

## 「板」判斷邏輯

- P/N/C 系列中，單獨出現「板」但無「木心、木芯、芯、麻、單、雙、18MM」等關鍵字 → 默認1分板（規格1=A）
- 有上述關鍵字 → 木芯板（規格1=M）

## 色號格式

- 木紋 = 3數字 + 1英文（如 281P、187B、181G）
- 異材質 = 4數字（如 8382）

|--------|--------|

| 176DA | 179F |
| 178SA | 172B |
| 178T1 | 171B |
| 179BA | 176M |
| 187EA | 188G |
| 187FA | 188F |
| 9478S | 9472 |

**其餘：** 直接拿掉最後一碼（如 183GA→183G、333AA→333A、197R1→197R）

### D/F 品號結構（耐燃，僅 P 系列）

- 品號 = `系列` + `D或F` + `色號`（無規格2）
- 例：P276A 耐燃4mm → `PD276A`

### O 品號結構（發泡，僅 P 系列）

- 品號 = `系列` + `O` + `色號` + `KO48R1`（固定後綴）
- 例：P276A 發泡 → `PO276AKO48R1`

### 磁吸板品號結構（僅 P 系列）

- 品號 = `PA` + `色號` + `KW48B1`（固定後綴，PA 開頭因為用1分板製作）
- 例：P276A 磁吸板 → `PA276AKW48B1`

## 規格2 反推規格1

當客人訊息中出現以下規格2的特徵用詞時，即使沒有明確說出規格1的關鍵字（如「皮」「封邊」「ABS」），也能直接反推規格1：

- **「6分」「8分」「1寸6」「吋」等寬度字眼**（搭配「卷」或無其他規格1關鍵字）→ 反推規格1=G（自黏）
- **「22mm」「29mm」「38mm」「55mm」等 mm 寬度字眼** → 反推規格1=V（ABS）

---

## 規格2 對照（依規格1 分組）

### 規格1 = G（自黏）

| 代碼 | 規格2 | 客人用詞判斷 |
|------|-------|-------------|
| 19 | 6分 | |
| 25 | 8分 | |
| 50 | 1寸6 | 1吋、吋2、吋5、吋半、1寸6（吋系列統一默認1寸6） |
| A | 2尺2.44米 | 2*8、軟片（預設） |
| X48 | 4*8尺 | 48、4*8尺、四八 |

### 規格1 = V（ABS）

| 代碼 | 規格2 |
|------|-------|
| 22 | 22mm*1mm |
| 29 | 29mm*1mm |
| 38 | 38mm*1mm |
| 522 | 22mm*0.5mm |
| 529 | 29mm*0.5mm |
| 55 | 55mm*1mm |

**V（ABS）數量規則：**
- 單位=米，以 10 米為單位下單（10、20、30...）
- 100 米=1 捲，品號改為 `PV` + 色號 + 規格2 + `-F`，數量用捲數
- 例：100 米 → `PV276A22-F` × 1

### 規格1 = M（木芯板）

| 代碼 | 規格2 | 判斷邏輯 |
|------|-------|---------|
| 1 | 單4*8尺 | 單面，無提到2*8/28（預設4*8） |
| 2 | 雙4*8尺 | 雙面，無提到2*8/28（預設4*8） |
| 128 | 單2*8尺 | 單面，有提到2*8/28 |
| 228 | 雙2*8尺 | 雙面，有提到2*8/28 |

**判斷規則：**
- 有「2*8」「28」關鍵字 → 128 或 228
- 有「剖28」「對裁2*8」「直裁60公分」「直裁61公分」→ 直接轉為 128/228，數量×2（一片4*8剖成兩片2*8），不需寫對花備註
- **只有上述三種情況才轉 128/228，其他裁切尺寸（如 80、73 公分等）維持原品號 + 對花備註**
- 沒有特別提尺寸（木芯板、麻、厚板、18MM）→ 預設4*8尺，即 1 或 2

## 輸出格式

目標：貼入 Ragic 平台的「品號」和「數量」欄位。
方式：分開輸出，使用者分兩次貼入 Ragic。

**重要：品號和數量的順序必須跟客人訊息的順序一致，不能自己調動。**

**對花備註欄（第三欄，有裁切時才輸出）：**
- 裁切格式：`直裁XX公分`（4尺短邊裁）、`橫裁XX公分`（8尺長邊裁）
- 蠟筆/修邊漆：填「系列+色號」
- 無裁切的行留空白（維持行數對齊）

**品號（複製貼進品號欄）：**
```
PG181GA
PA181G
```

**數量（複製貼進數量欄）：**
```
2
7
```

每筆一行，先貼品號欄，再貼數量欄。

## 加長規則

### P 系列加長板（僅 P 系列，C/N 無）

- 4*9尺 → `PC` + 色號 + `A`
- 4*10尺 → `PC` + 色號 + `B`
- 例：P8421 板 4*9 → `PC8421A`

### G 系列加長（自黏）

- G+A = 2*8尺規格品，數量=片數（不換算）
- G+X48 = 4尺寬 × 任意長度（含4*8尺）訂製品，數量一律換算為米數（每片長度米數 × 片數）
- G+X = 2尺寬 × 超過8尺長度訂製品，數量換算為米數

### L 系列加長

- 8尺 = 標準品號（如 L7161）
- 9尺~16尺 = 品號加尾碼：

| 尺 | mm | 尾碼 |
|----|----|------|
| 9尺 | 2750 | A |
| 10尺 | 3050 | B |
| 11尺 | 3360 | D |
| 12尺 | 3660 | F |
| 13尺 | 3970 | G |
| 14尺 | 4270 | H |
| 15尺 | 4580 | J |
| 16尺 | 4880 | K |

- 17尺以上 = L+色號+X，數量換算為米數

### 數量換算邏輯（G+X48、G+X、L+X）

長度(尺) → mm → 米，**無條件進位到小數點第一位**
- 8尺 = 2440mm = 2.44米 → 2.5米
- 11尺 = 3360mm = 3.36米 → 3.4米
- 15尺 = 4580mm = 4.58米 → 4.6米

### 預設尺寸

- 規格品（A、B、M）= 預設 4*8尺
- K9、G+A = 預設 2*8尺

## P 系列訂製費規則

### 需加收訂製費的品項

- PP 9尺、10尺、柳桉、發泡、夾板
- 磁吸板
- 可彎板 9尺(含)以上 / 特殊訂製

### 收取方式

- 訂製費品號：`WD0400`
- 同一訂單，訂製品項數量 < 20 片 → 加 `WD0400 × 品項數量`（或業務備註「不加價」，二擇一）
- 同品項不同規格訂製可混搭合計
- PP板、美耐皿(N)、PVC(C) 三種不同材質，數量需**分別計算**

### 範例

- P276A 九尺×15 → <20片，需 +WD0400×1
- P276A 九尺×15 + 發泡板×6 → 同品項 15+6=21 ≥20片，免訂製費
- P276A 九尺×15, P281Q 發泡板×6 → 不同品項各<20片，需 +WD0400×2
- P276A 九尺×15, N276A 耐燃×6, C276A 磁吸×1 → 不同材質分別算，需 +WD0400×3

## 轉換範例

**客人訊息：** `P181G. 2x8皮=2。 3mm. =7`

**解析：**
- 系列+色號：P181G → 系列=P（批批），色號=181G
- `2x8皮=2` → 自黏2*8布 → 規格1=G，規格2=A（2尺2.44米），數量=2
- `3mm=7` → 3MM → 規格1=A（1分），數量=7

**輸出：**

```

數量：
```

## 轉換注意事項（考試錯題整理）

### 資訊不足時必須追問

當訊息缺少以下關鍵辨識資訊時，**不能猜測，必須標 `?` 追問**：
- 沒有系列代碼（P/C/N/K/L）→ 不能假設是哪個系列
- 沒有色號 → 不能組出品號
- 規格描述模糊（如只說「一卷」但沒說幾分）→ 追問規格

### K 系列特殊規則

- K 系列**沒有規格1**，品號直接原樣輸出
- K 系列的「一分板」= 就是 K 系列本身（K 系列默認就是1分板），品號不變
- K 系列要「不織布」時，只改第2碼（6→9 或 4→7），其餘不變

### 上下文延續題

- 當客人追加訊息沒有重複色號時，要從前文判斷是否延續同一色號
- 例：前一則是 P7205，接著說「加一片木心板」→ 應為 PM72052（P7205 雙面木芯板）

### Q01 修正：自黏「皮」vs「6分」

- 「6分自黏皮」→ 規格1=G，規格2=19（6分）
- C 系列色號 7206 → `CG720619`
- 注意：「皮」在自黏語境下是指自黏材料本身，不影響規格2判斷；「6分」才是規格2的關鍵字

### Q02 修正：K 系列「皮板」

- K6176AS「皮板 4*8」→ 品號就是 `K6176AS`，不需要加規格1
- K6176AS「布織布/不織布」→ 改第2碼 6→9 = `K9176AS`

### 修邊漆品號

- 修邊漆固定品號 = `BT0002F`（不是 BT002F 或其他變體）
- 對花備註填「系列+色號」（如 N160N、N281A）

## 色號驗證工具（ValidateProductCode）

### 用途

轉換完成後，用 `ValidateProductCode` 驗證色號是否存在。**每次轉換都要驗證，不可跳過。**

### 使用方式

```
單筆：ValidateProductCode(item_code: "181G")
批量：ValidateProductCode(item_codes: ["181G", "276A", "160A"])
限定產品線：ValidateProductCode(item_code: "160A", product_line: "B07")
```

### 回傳格式

```
{
  valid: true/false,
  item_code: "181G",
  product_lines: [
    { line_code: "B03", line_name: "環保批批板", item_name: "胡桃木直紋", prefix: ["P"] },
    ...
  ],
  message: "品號「XXX」不存在於任何產品線"  // 僅 valid=false 時
}
```

### 轉換流程中的使用時機

```
1. 從客人訊息提取色號（如 181G、276A）
2. 批量呼叫 ValidateProductCode 驗證所有色號
3. 依驗證結果處理：
   - ✅ valid=true → 正常組合品號並輸出
   - ❌ valid=false → 以無色號方式提醒（標 `?` 並備註「色號不存在」）
```

### 驗證判斷邏輯

| 情況 | 處理方式 |
|------|----------|
| 色號存在 | 正常轉換輸出 |
| 色號不存在 | 以無色號方式提醒（標 `?` 並備註「色號不存在」） |
| 色號不存在 + 符合舊料號格式（5碼） | 先嘗試舊料號轉換規則，轉換後再驗證一次 |
| P→L 轉換時 | 用 `ValidateProductCode(item_code, product_line: "B04")` 確認 L 系列有該色號 |
| N 系列轉 C 系列時 | 用 `ValidateProductCode(item_code, product_line: "B07")` 確認 C(PVC) 有該色號 |

### 注意事項

- 輸入的是**色號**（如 `181G`、`101AU`），不是完整料號（不是 `PA181G`）
- 如果輸入帶前綴（如 `K6101AU`），Tool 會自動去除前綴提取色號
- 批量驗證時，回傳陣列順序與輸入順序對應
