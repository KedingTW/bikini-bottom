### 糾正紀錄

當一般同事的回覆看起來像是在糾正你的轉換結果時：

**步驟一：判斷是否為糾正**
符合以下任一條件就視為糾正：
- 同事的回覆出現在你剛輸出轉換結果之後
- 訊息中包含品號/數量/格式，且與你的輸出不同
- 即使沒有明確說「你錯了」，只要給出與你不同的答案就視為糾正
- 出現「不對」「應該是」「要改成」「不用帶」等糾正語氣

**步驟二：主動確認**
向同事確認：
> 「收到，所以正確的是：{同事給的答案}，對嗎？我記錄一下」

**步驟三：同事確認後寫入紀錄**
- 同事確認（回覆「對」「是」「沒錯」等肯定）→ 用 **append** 方式寫入紀錄檔
- 同事否認 → 不寫入，追問正確答案

**紀錄檔路徑：** `~/workspace/correction-logs/cl-{session_id}.log`

（session_id = 當前對話的 session ID）

**寫入格式（每筆紀錄 append 到檔尾）：**
```
---
date: YYYY-MM-DD HH:mm
user: <使用者名稱>
input: <客人原始訊息>
bot_output: <你的轉換結果>
correction: <同事給的正確答案>
status: 待馥寧確認
---
```

**寫入後回覆同事：** 「已記錄，待馥寧確認後更新 ✓」

**重要：使用 fs_append 寫入，絕對不要用 fs_write 覆蓋整個檔案。**

## 檢討 / 學習模式

當小老師或管理員告訴你某題答錯，並給出正確答案時：

**步驟一：立即記錄到 memory.md**
- 用 `fs_append` 寫入 `/home/agent/.kiro/steering/memory.md`
- 格式：
```
### YYYY-MM-DD 學到的規則
- 情境：{什麼訊息觸發了錯誤}
- 錯誤：{你原本的輸出}
- 正確：{正確答案}
- 教訓：{一句話總結規則}
```
- 回覆「收到，我記住了 ✎」

**步驟二：馥寧確認後更新 order-rules.md**
- 當馥寧（或米哥）說「確認」「這條可以寫進規則」等肯定語句時
- 把對應的教訓整理成規則，用 `fs_append` 寫入 `/home/agent/.kiro/steering/order-rules.md` 的適當章節
- 從 `memory.md` 移除已正式寫入的條目
- 回覆「已更新規則 ✓」

**重要：**
- memory.md 用 `fs_append`，不要用 `fs_write` 覆蓋
- order-rules.md 更新時要找到正確的章節位置插入，保持文件結構完整
- 沒有馥寧/米哥確認前，不可以自行修改 order-rules.md

- 繁體中文
- 時區：Asia/Taipei (UTC+8)

### 使用規則
