# 🗄️ 共用儲存規範（科定AI）

## 掛載路徑

| 容器內路徑 | 權限 | 用途 |
|-----------|------|------|
| `/mnt/kd-share` | **唯讀** | 公司共用檔案（使用者貼路徑→你去讀） |
| `/home/agent/workspace` | **可讀寫** | 你的個人工作區（projects 等） |
| `/shared/workspace` | **可讀寫** | 科定AI 團隊共用工作區（跨角色協作） |

## Windows 路徑轉換

使用者的 `Z:\` 磁碟機 = 容器內 `/mnt/kd-share`。

**轉換規則：**
- `Z:\任何路徑` → `/mnt/kd-share/任何路徑`
- 將 `\` 替換為 `/`
- 去掉開頭的 `Z:`

**範例：**
- `Z:\03_生產單位\02-產銷統合\報表.xlsx` → `/mnt/kd-share/03_生產單位/02-產銷統合/報表.xlsx`

**不需要問使用者確認，不需要搜尋，直接轉換後讀取。**
**不要對使用者解釋掛載原理或技術細節。**

## ⚠️ 絕對規則

### `/mnt/kd-share`（公司共用區）

| 操作 | 允許？ | 說明 |
|------|--------|------|
| 讀取使用者指定的完整路徑檔案 | ✅ | 使用者貼路徑，你去讀 |
| `find`、`ls -R`、遞迴搜尋 | ⛔ **禁止** | 檔案數量極龐大，會造成系統負載 |
| `ls` 列出某層目錄 | ⚠️ 僅限使用者明確要求時 | 最多列一層，禁止遞迴 |
| 寫入任何檔案 | ⛔ **禁止** | 唯讀，你本來就不該寫 |
| `grep -r` 搜尋檔案內容 | ⛔ **禁止** | 龐大目錄 |

### `/home/agent/workspace`（個人工作區）

| 操作 | 允許？ |
|------|--------|
| 讀取、搜尋 | ✅ |
| 寫入、建立檔案 | ✅ |
| `find`、`ls -R` | ✅ |

### `/shared/workspace`（團隊共用工作區）

| 操作 | 允許？ |
|------|--------|
| 讀取、搜尋 | ✅ |
| 寫入、建立檔案 | ✅ |
| `find`、`ls -R` | ✅ |

## 使用流程

```
使用者說：「幫我看 Z:\03_生產單位\02-產銷統合\報表.xlsx」

→ 轉換路徑：/mnt/kd-share/03_生產單位/02-產銷統合/報表.xlsx
→ 直接讀取該檔案
→ 如需產出結果，寫到 ~/workspace/ 或 /shared/workspace/ 下
```

## 檔案放哪？

| 情境 | 放哪 |
|------|------|
| 自己的草稿、進行中的工作 | `~/workspace/projects/` |
| 要交付或跨角色協作 | `/shared/workspace/` |
| 讀公司共用檔案 | `/mnt/kd-share/`（唯讀） |

## 錯誤處理

- 如果路徑不存在：告知使用者「找不到該檔案，請確認路徑是否正確」
- **永遠不要**嘗試搜尋來「幫忙找」檔案可能的位置
- **永遠不要**嘗試對 `/mnt/kd-share` 寫入
