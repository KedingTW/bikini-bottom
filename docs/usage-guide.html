📋 步驟說明

1
交辦任務
在 Discord #general 頻道 @mention 對應的工程師，描述你要的功能。
2
等待開發
工程師會確認需求、寫程式碼、開 PR、呼叫泡芙老師審核。你不需要做任何事。
3
自動審核
泡芙老師檢查程式碼品質。有問題會通知工程師修正（最多 3 輪），全自動。
4
收到通知
審核通過後，泡芙老師會在 Discord @mention 你，附上 PR 連結。
5
合併 PR
點開連結確認沒問題，按 Merge 按鈕。完成！

💬 交辦範例

方式 A：Discord 直接交辦

你：
@海綿寶寶 在 krusty-krab-website 新增一個「聯絡我們」頁面，
要有表單（姓名、Email、訊息），送出後顯示感謝訊息。

海綿寶寶：
收到！我來確認一下需求...
（接著自動開發 → 開 PR → 呼叫泡芙老師）

方式 B：Redmine Issue 交辦

你（在 Redmine 建好 Issue #1234 後）：
@海綿寶寶 請處理 Redmine Issue #1234

海綿寶寶：
收到！讓我讀取 Issue 內容...
（讀取 Issue → 更新狀態為「開發中」→ 開發 → 開 PR → 審核 → 更新狀態為「已完成」）

交辦 Tips

✅ 說清楚是哪個專案（repo 名稱）
✅ 描述你要的功能，越具體越好
✅ 前端找海綿寶寶，後端找派大星
❌ 不需要告訴他怎麼寫程式碼
❌ 不需要跟泡芙老師說話（她自動工作）

⚠️ 注意事項

1️⃣ 一次一個任務 — 等上一個 PR 合併後再交辦下一個
2️⃣ 指定專案 — 告訴工程師是哪個 repo
3️⃣ 合併由你操作 — AI 不會自己合併 PR，最終決定權在你
4️⃣ 3 輪上限 — 修了 3 輪還沒通過，泡芙老師會通知你來決定

❓ 常見問題

Q: 工程師沒回應怎麼辦？
等 30 秒看有沒有 👀 反應。沒有的話再 mention 一次。
Q: 我想改需求怎麼辦？
直接在同一個 thread 裡跟工程師說，他會調整。
Q: 泡芙老師太嚴格了？
她只會擋有 bug 或安全問題的程式碼。建議和備註不會擋通過。
Q: 可以同時交辦給兩個工程師嗎？
可以，但建議是不同專案的任務。同一個專案同時改容易衝突。
